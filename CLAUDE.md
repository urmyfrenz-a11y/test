# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

마크다운(`.md`) 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트입니다.
글쓴이는 마크다운으로 글을 작성하고, 이 프로젝트가 이를 읽기 좋은 HTML 블로그로 렌더링합니다.

**핵심 목표**
- 깔끔하고 읽기 좋은 디자인 (콘텐츠 가독성 우선)
- 다크 모드 지원 (시스템 설정 연동 + 수동 토글)
- 모바일 반응형 (모든 화면 크기에서 잘 보이도록)

## 기술적 제약

- **프레임워크 없음.** React, Vue, Svelte, Next.js 등 사용하지 않습니다.
- **순수 HTML, CSS, JavaScript(바닐라)로만** 구현합니다.
- 빌드 도구 없이 브라우저에서 바로 동작하는 것을 기본으로 합니다.
  (마크다운 파싱이 필요하면 CDN 기반의 경량 라이브러리 또는 직접 구현한 파서를 사용)
- 외부 의존성은 최소화합니다. 추가가 필요하면 먼저 이유를 설명하세요.

## 권장 구조

아직 코드가 없는 초기 단계입니다. 아래 구조를 기본 방향으로 삼되, 작업 중 더 나은 안이 있으면 제안하세요.

```
my-blog/
├── index.html          # 글 목록 (홈)
├── post.html           # 개별 글 보기 (마크다운 렌더링)
├── css/
│   └── style.css       # 스타일 + 다크 모드 + 반응형
├── js/
│   ├── markdown.js     # 마크다운 → HTML 파싱
│   ├── theme.js        # 다크 모드 토글 / 저장
│   └── app.js          # 글 목록 로드, 라우팅
└── posts/
    ├── posts.json      # 글 메타데이터 목록 (제목, 날짜, 파일명, 요약)
    └── *.md            # 실제 글 마크다운 파일
```

## 디자인 원칙

- **가독성 우선**: 본문 폭 제한(약 65~75자), 충분한 줄 간격(line-height 1.6~1.8), 편안한 폰트 크기(본문 16~18px).
- **색상은 CSS 변수로 관리**: 라이트/다크 테마를 `:root`와 `[data-theme="dark"]`의 CSS 커스텀 속성으로 전환.
- **다크 모드**:
  - `prefers-color-scheme`로 시스템 설정을 기본값으로 따름.
  - 수동 토글 버튼 제공, 선택값은 `localStorage`에 저장.
  - 페이지 깜빡임(FOUC) 방지 — 테마는 가능한 한 빨리 적용.
- **반응형**: 모바일 우선(mobile-first)으로 작성, `max-width` 미디어 쿼리로 보강. 터치 타깃 최소 44px.
- **코드 블록**: 가로 스크롤 가능, 다크 모드에서도 대비 유지.

## 글 추가 방법

1. `posts/` 폴더에 `<slug>.md` 파일을 작성합니다. (예: `posts/my-first-post.md`)
2. 본문 맨 위에 제목을 `# `로 다시 쓰지 마세요 — 제목은 `posts.json`의 `title`로
   글 페이지 헤더에 표시되므로 중복됩니다. 본문은 소제목(`##`)부터 시작하세요.
3. `posts/posts.json` 배열에 항목을 추가합니다:
   ```json
   {
     "slug": "my-first-post",
     "title": "내 첫 글",
     "date": "2026-06-24",
     "summary": "목록에 보일 한 줄 요약"
   }
   ```
   - `slug`은 `.md` 파일명(확장자 제외)과 일치해야 합니다.
   - 목록은 `date` 기준 최신순으로 자동 정렬됩니다.

## 작업 규칙

- 새 기능을 추가할 때 기존 파일/스타일의 컨벤션(들여쓰기, 네이밍, 주석 밀도)을 따르세요.
- CSS는 변수 기반으로 작성하고, 하드코딩된 색상값을 흩어두지 마세요.
- JavaScript는 모듈 단위로 역할을 분리하고, 전역 오염을 피하세요.
- 시맨틱 HTML(`<article>`, `<nav>`, `<main>`, `<time>` 등)과 접근성(alt, aria, 키보드 포커스)을 지키세요.

## 실행 / 미리보기

정적 사이트이므로 로컬 서버로 확인합니다 (파일 fetch가 `file://`에서 막힐 수 있으므로 서버 권장):

```bash
# Python
python -m http.server 8000

# 또는 Node
npx serve .
```

브라우저에서 `http://localhost:8000` 접속.
