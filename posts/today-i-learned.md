클로드 코드로 마크다운 블로그를 직접 만들면서, 웹페이지를 이루는 세 가지 기술이 각각 어떤 일을 하는지 감이 잡혔습니다. 오늘 배운 것을 간단히 정리해 둡니다.

## HTML — 뼈대(구조)

HTML은 페이지의 **구조와 내용**을 담당합니다. 제목, 문단, 목록, 링크, 이미지처럼 "무엇이 있는지"를 정의하는 역할입니다.

이 블로그에서는 글 제목을 `<h1>`, 본문을 `<article>`, 글 목록을 `<ul>`로 표현했습니다. 의미에 맞는 태그(`<article>`, `<nav>`, `<time>` 등)를 쓰면 화면에 보이는 모습뿐 아니라 검색엔진과 화면낭독기에게도 내용의 의미가 전달됩니다.

> HTML만 있으면 장식 없는 문서 한 장처럼 보입니다. 내용은 다 있지만 꾸며지지 않은 상태죠.

## CSS — 옷(디자인)

CSS는 HTML이 만든 뼈대에 **모양과 색을 입히는** 역할입니다. 글자 크기, 색상, 여백, 배치, 반응형 레이아웃이 모두 CSS의 몫입니다.

이번에 특히 유용했던 건 **CSS 변수**였습니다. 색을 한 곳에 정의해두면 라이트/다크 모드를 손쉽게 전환할 수 있었습니다.

```
:root {
  --bg: #ffffff;
  --text: #1f2328;
}
[data-theme="dark"] {
  --bg: #0d1117;
  --text: #e6edf3;
}
```

또 미디어 쿼리로 화면 폭에 따라 글자 크기와 여백을 조절해서, 모바일에서도 읽기 좋게 만들 수 있었습니다.

## JavaScript — 움직임(동작)

JavaScript는 페이지에 **동작과 상호작용**을 더합니다. 버튼 클릭에 반응하고, 데이터를 불러오고, 화면을 바꾸는 일을 합니다.

이 블로그에서 JS가 한 일은 크게 세 가지였습니다.

1. **마크다운 변환** — `.md` 파일을 읽어 HTML로 바꿔 화면에 그려줍니다.
2. **글 목록 불러오기** — `posts.json`을 `fetch`로 가져와 목록을 만듭니다.
3. **다크 모드 토글** — 버튼을 누르면 테마를 바꾸고 `localStorage`에 기억해 둡니다.

## 정리

| 기술 | 역할 | 비유 |
| ---- | ---- | ---- |
| HTML | 구조·내용 | 뼈대 |
| CSS | 디자인·배치 | 옷 |
| JavaScript | 동작·상호작용 | 움직임 |

셋이 각자의 역할을 나눠 맡고 협력하면, 프레임워크 없이도 다크 모드와 모바일 대응을 갖춘 블로그를 만들 수 있다는 걸 직접 확인했습니다. 다음엔 글 검색 기능에 도전해봐야겠습니다.
